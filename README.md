# NGFW Manager

Веб-интерфейс для управления политиками безопасности NGFW.  
Позволяет организовывать правила в виртуальные папки, создавать, копировать и переносить правила между устройствами — не затрагивая структуру самого NGFW.

---

## Быстрый старт

### 1. Клонировать / скачать проект

```bash
git clone <repo-url>
cd ngfw_manager
```

### 2. Настроить `.env`

Файл `.env` уже есть в корне. При необходимости измените название:

```env
APP_TITLE="NGFW Manager"
```

### 3. Сгенерировать SSL-сертификаты

```bash
bash init_ssl.sh
```

Создаст самоподписанные сертификаты в `nginx/certs/`. Если у вас есть валидные сертификаты — положите их туда как `ngfw.crt` и `ngfw.key`.

### 4. Запустить

```bash
docker compose up -d --build
```

Приложение будет доступно по адресу: **https://localhost**

> При первом запуске Docker скачает образы и соберёт контейнер — это займёт 1–2 минуты.

### 5. Остановить

```bash
docker compose down
```

## Структура контейнеров

| Контейнер | Роль |
|-----------|------|
| `ngfw_backend` | FastAPI-приложение (порт 8000 внутри сети) |
| `ngfw_proxy` | Nginx — HTTPS-прокси, порты 80 и 443 |
| `ngfw_manager-db-1` | PostgreSQL 15 — локальная база данных |

## Вход в систему

Откройте **https://localhost** в браузере.

На странице входа заполните:
- **NGFW Host / URL** — адрес вашей системы управления, например `https://192.168.1.100:3030` для запуска приложения из контейнра необходимо ввести  `https://host.docker.internal:3030` 
- **Username** — имя пользователя API
- **Password** — пароль

Сессия сохраняется в cookie браузера.

## Основной функционал

### Синхронизация (Sync)

Кнопка **Sync** в верхней панели загружает с устройства:
- Все группы устройств (device groups)
- Все объекты: сети, сервисы, зоны, приложения, пользователи
- Все правила безопасности (Pre / Default / Post)

После синхронизации правила распределяются по виртуальным папкам. Структура папок хранится только локально — NGFW её не видит.

### Виртуальные папки

В боковом сайдбаре можно:
- Создать папку (кнопка **+** рядом с разделом Pre / Default / Post)
- Перетащить правила между папками (drag-and-drop)
- Выделить несколько правил и переместить их в папку через **Move to Folder**

### Создание правила

Кнопка **+ New Rule** открывает форму:
- Название, описание, действие (Allow / Drop / Deny)
- Зоны источника и назначения
- Сетевые объекты источника и назначения
- Сервисы
- Включено / выключено

Объекты выбираются через пикер с живым поиском. Поддерживается мультиселект.

### Копирование и перенос правил

1. Выделите одно или несколько правил (чекбоксы)
2. Нажмите **Transfer / Copy** в нижней панели
3. Выберите целевое устройство и папку
4. Нажмите **Copy** (оригинал остаётся) или **Move** (оригинал удаляется)

При копировании объекты автоматически:
- **Глобальные** — используются как есть (UUID совпадает на всех устройствах)
- **Устройство-специфичные** — ищутся на целевом устройстве по значению (fingerprint); если не найдены — создаются копии с суффиксом `_copy`

### Страница Objects

Кнопка **Objects** в верхней панели открывает справочник всех объектов.

- Выбор устройства — в боковом сайдбаре
- Фильтрация по типу: Networks / Services / Zones / Applications / Users
- Поиск по имени — в строке над таблицей
- Глобальные объекты помечены бейджем **GLOBAL**

### Deploy

Кнопка **Deploy** применяет текущий порядок правил из виртуальных папок к NGFW через `MoveSecurityRule`.

## Обновление приложения

```bash
# Остановить контейнеры
docker compose down

# Скопировать новые файлы в директорию проекта

# Пересобрать и запустить
docker compose up -d --build
```

Если обновлялся `sync_service.py` — после запуска рекомендуется очистить кэш объектов и сделать повторный Sync.
